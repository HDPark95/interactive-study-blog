# 기술 문서 작성 스킬

기술 문서를 작성하거나 리뷰할 때 사용하는 스킬입니다.
3단계 프로세스(문서 유형 → 정보 구조 → 문장 다듬기)를 적용합니다.

---

## Step 1: 문서 유형 정하기

문서 작성 전, 다음을 먼저 파악합니다:
- 문서의 목표는 무엇인가?
- 독자는 누구인가? (초급/중급/고급)
- 어떤 상황에서 읽히는가?

### 문서 유형별 체크리스트

**학습용 (Tutorial)**
- [ ] 명확한 학습 목표 제시
- [ ] 완료 후 얻게 될 능력 명시
- [ ] 단계별 진행 구조
- [ ] 실습 예제 포함

**문제 해결용 (How-to)**
- [ ] 명확한 문제 상황 정의
- [ ] 단계별 해결 방법 제시
- [ ] 전제 조건 명시
- [ ] 예상 결과 제시

**참조용 (Reference)**
- [ ] 간결한 개요
- [ ] 빠른 탐색 가능한 구조
- [ ] 사용 예제 코드 포함
- [ ] 파라미터/옵션 설명

**설명용 (Explanation)**
- [ ] 기술이 등장한 배경
- [ ] 핵심 개념 설명
- [ ] 왜 이렇게 동작하는지
- [ ] 실제 사용 사례

---

## Step 2: 정보 구조 만들기

### 5가지 핵심 원칙

1. **한 페이지, 하나의 목표**
   - 여러 목표가 섞이면 분리
   - 독자가 한 가지에 집중할 수 있게

2. **개요 필수**
   - 첫 문단에서 무엇을 다루는지 요약
   - 독자가 이 문서가 자신에게 필요한지 빠르게 판단

3. **예측 가능한 구조**
   - 일관된 헤딩 레벨 사용
   - 비슷한 내용은 비슷한 형식으로

4. **가치 먼저**
   - 결론/핵심을 먼저 제시
   - 배경 설명은 나중에

5. **효과적인 제목**
   - 내용을 정확히 반영
   - 검색 가능한 키워드 포함
   - 행동 동사 사용 (설정하기, 이해하기)

---

## Step 3: 문장 다듬기

### 5가지 문장 원칙

1. **필요한 정보만**
   - 불필요한 수식어 제거
   - "~라고 할 수 있습니다" → "~입니다"
   - 중복 표현 제거

2. **메타 담화 최소화**
   - "이 섹션에서는 ~를 설명합니다" 제거
   - 바로 본론으로

3. **구체적으로**
   - "빠르게" → "100ms 이내에"
   - "여러 개" → "최대 10개"
   - 모호한 표현 지양

4. **일관되게**
   - 같은 개념은 같은 용어로
   - 문체 통일 (합쇼체/해요체)
   - 코드 스타일 통일

5. **주어 명확히**
   - 누가 행동하는지 분명하게
   - "설정해야 합니다" → "개발자가 설정해야 합니다" 또는 "설정하세요"

---

## 적용 방법

문서 초안을 작성한 후:

1. **유형 점검**: 이 문서의 유형에 맞는 체크리스트 확인
2. **구조 점검**: 5가지 구조 원칙 적용
3. **문장 점검**: 5가지 문장 원칙으로 다듬기

### 리뷰 요청 시

```
다음 문서를 tech-writing 스킬로 리뷰해주세요:
- 문서 유형: [학습용/문제해결/참조/설명]
- 대상 독자: [초급/중급/고급]

[문서 내용]
```
